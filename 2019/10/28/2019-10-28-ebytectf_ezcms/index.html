以前被buuoj平台打到心态爆炸，现在再一次做题，争取每道题都自己耐心做，做不下了再看writeup， 毕竟buu收录的题目质量真的高，这样才能学到东西

首先是代码泄漏，不过buuoj如果短时间创建几个新请求会返回429,请求过快，所以扫描要单线程，使用requests.session(),写一个扫一下

然后代码审计，看着很清晰，有一个config.php，里面有很多类，而且还看到了__call和__destruct这两个魔术方法，这样大概率就是phar扩展的反序列化和pop链的构造。
config.php 还有一个验证是不是admin身份。

很明显用的是hash长度拓展攻击
这里就弄了两个小时，在本地一下成功了，但是在靶机上面一直失败，无奈之下下个题目的docker镜像来实现，得到一个吐血的结论，buu这道题目靶机出问题了。。。。。
使用hashpump或者python的hashpumpy，这里我使用了python
python代码没保存就不放代码了。

这里有个上传的功能，没有什么过滤，创建一个文件夹然后上传到这个文件夹上面，但是config.php 自动生成一个.htaccess文件，一访问就会返回500错误，所以上传了shell也没用，这个应该是.htaccess 的错误引起的
然后看到view.php上面

有个mime_content_type这个应该是可以触发phar的，但是正则不能phar开头，但是没有禁php，可以绕过，使用
php://filter/resource=phar://xxxxx
或者php://filter/read=convert.base64-encode/resource=phar://xxxx等等来绕过

然后构造pop链

这里我设想的是利用config.php中的upload_file()来上传文件到这个文件夹以外的地方，然后就不会受到.htaccess的影响了，
然后生成phar文件的phar.php也构造出来了，在本地测试也成功触发phar了，但是无法上传，这时看到move_uploaded_file的php文档，
发现它会检测临时文件是不是上传的文件，然后才会执行，如果要移动的文件不是本次上传的文件，就会返回false，然后设想将$this->file_tmp =一个临时文件名，来爆破临时文件名，可惜要爆破61的5次方次才可能实现，洗个澡回来生成了2G的缓存，
真的是自己ddos自己。。。。。。没办法去看writeup了

原来是有一个ZipArchive::open 的函数，
ZipArchive::open的官方描述是这样的
ZipArchive::open ( string $filename [, int $flags ] ) : mixed
打开一个新的zip存档以进行读取，写入或修改。，所以就可以修改.htaccess 文件了

config.php 中Profile类有一个
__call魔术方法，是这样的

所以让$this->admin= 类ZipArchive就可以了
让$this->username= 文件的地址.htaccess
$this->password = 要修改成的内容

然后生成phar文件，再上传文件，然后在view.php?filepath=php://filter/read=convert.base64-encode/resource=phar://./sandbox/3b1412753f475cc969c37231dd6eaea2/9c7f4a2fbf2dd3dfb7051727a644d99f.phar

类似这样就可以了

总结：

在看到诡异的__call方法利用的时候，而且代码中没有什么类可以利用的时候，要想到php内置类，这样谷歌搜 “::open” Manual-PHP
寻找内置化类这方面还真不咋会
